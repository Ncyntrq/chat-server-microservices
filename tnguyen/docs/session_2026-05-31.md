## 4. Xử lý security review (3 HIGH)
Lỗi: gửi Bearer token qua URL / tới host do server cấp (exfiltration nếu attacker điều khiển `iconUrl`/`avatarUrl`).

- **Xóa `FileApiClient.authedUrl`** (đặt token trong `?token=` → lộ qua log/history).
- Thêm `FileApiClient.download(fullUrl)`: JWT qua **header Authorization**, chỉ gửi khi `isTrustedGatewayUrl()` khớp host/scheme/port gateway.
- `ChatMessageItem`: preview ảnh + tải file đều qua `download()`; tải file lưu bằng `JFileChooser` (bỏ `Desktop.browse` + token-in-URL).
- `AvatarBadge.loadAvatarFromUrl` & `ServerIconItem.loadServerIconFromUrl`: route qua `download()` → có guard host. URL ngoài bị từ chối, không gửi token.
